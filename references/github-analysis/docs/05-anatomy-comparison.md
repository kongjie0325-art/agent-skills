## 一、本质区别

### 一句话本质

| 能力 | 一句话本质 | 解决什么问题 |
|------|-----------|-------------|
| **Skills** | 知识+流程模板 | 告诉 Agent **怎么做** |
| **MCP** | 工具调用标准协议 | 让 Agent **能做** |
| **Plugin** | 封装好的功能扩展 | **增加**某个能力 |
| **Memory** | 持久化上下文 | 让 Agent **记住** |
| **Hooks** | 生命周期拦截器 | 让 Agent **安全地做** |

### 没有会怎样？

```
用户说：帮我分析 GitHub 项目并部署到 VPS

没有 Skills：
  Agent 知道要部署，但不知道最佳流程
  → 像实习生，能干活但没经验

没有 MCP：
  Agent 知道流程，但无法操作 GitHub
  → 像高级工程师，有经验但没工具

没有 Plugin：
  Agent 能操作 GitHub，但无法调用特殊服务
  → 基础能力齐全，但缺生态集成

没有 Memory：
  Agent 每次都从零开始
  → 没有连续性，重复解释

没有 Hooks：
  Agent 可能执行危险操作
  → 没有安全门卫
```

| 维度 | Skills | MCP | Plugin | Memory | Hooks |

| **智能性提升** | ★★★★★ | ★★★ | ★★ | ★★★★ | ★ |

| **扩展性** | ★★★★★ | ★★★★★ | ★★ | ★★★ | ★★★★ |

| **长期价值** | 极高 | 极高 | 中 | 极高 | 高 |

### 按"谁最重要"排序

```
1. Skills     → 决定上限（质量）
2. MCP        → 决定行动力（执行力）
3. Memory     → 决定连续性（记忆）
4. Hooks      → 决定安全性（治理）
5. Plugin     → 决定功能补充（生态）
```

## 三、决策树

```
我需要给 Agent 添加新能力
│
├─ 是外部工具/系统连接？
│   ├─ 是 → 需要 MCP
│   │   ├─ 多 Agent 复用？ → 独立 MCP Server
│   │   └─ 仅当前 Agent？ → Function Calling
│   │
│   └─ 否 ↓
│
├─ 是知识/流程/最佳实践？
│   │   ├─ 是 → 需要 Skill
│   │   │   ├─ 需要外部工具？ → Skill + MCP
│   │   │   └─ 不需要？ → 纯 Skill
│   │   │
│   │   └─ 否 ↓
│   │
├─ 是安全/治理/拦截？
│   │   ├─ 是 → 需要 Hook
│   │   │
│   │   └─ 否 ↓
│   │
├─ 是给非技术团队用？
│   │   ├─ 是 → 需要 Plugin（打包 Skill+MCP+Hook）
│   │   │
│   │   └─ 否 ↓
│   │
├─ 是需要记住的信息？
│   │   ├─ 是 → Memory（CLAUDE.md / Vector DB）
│   │   │
│   │   └─ 否 → 可能不需要任何扩展
```

## 四、类比总结

### 建房子类比

| 能力 | 类比 | 说明 |
|------|------|------|
| **Skills** | 建筑图纸 | 告诉工人怎么建 |
| **MCP** | 电动工具 | 让工人能干活 |
| **Plugin** | 预制件 | 直接装上就能用 |
| **Memory** | 施工日志 | 记住之前做了什么 |
| **Hooks** | 安全检查 | 每一步都检查安全 |
| **LLM** | 工人 | 实际干活的人 |

### 人脑类比

| 能力 | 类比 |
|------|------|
| **Skills** | 经验和知识（知道怎么做）|
| **MCP** | 手和脚（能做）|
| **Plugin** | 特殊工具（显微镜、计算器）|
| **Memory** | 记忆（长期记忆）|
| **Hooks** | 条件反射/习惯（自动反应）|
| **LLM** | 大脑本身 |

## 五、能力提升量化

虽然不是精确量化，但符合多数高级 Agent 实际体验：

```
仅模型                    ≈ 1x
模型 + Plugin             ≈ 2x
模型 + MCP                ≈ 5x
模型 + Skills             ≈ 8x
模型 + Skills + MCP + Memory ≈ 20x+
```

### 为什么 Skills 的乘数效应最大？

1. **Skills 决定推理质量** — 同样的模型，有 Skill 像高级工程师，没 Skill 像实习生
2. **Skills 是"杠杆"** — 一次编写，每次推理都提升质量
3. **Skills 是"经验"** — 编码了人类专家的知识

### 为什么 MCP 排第二？

1. **MCP 决定行动力** — 没有工具，再好的推理也白搭
2. **MCP 是"执行力"** — 从"说"到"做"的跨越
3. **MCP 是"生态"** — 连接一切外部系统
