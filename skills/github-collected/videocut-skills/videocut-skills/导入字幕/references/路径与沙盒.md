# macOS 沙盒与路径踩坑

## 核心问题

macOS 剪映是沙盒 App，**只能读 `~/Movies/JianyingPro/` 下自己的目录**。
把草稿写到 `/tmp`、`/private/tmp`、桌面等位置，打开时一律"媒体丢失"。

## 解决：DRAFT_DIR 改到 Projects

`setup.sh` 会自动把 `capcut-mate/config.py` 的 DRAFT_DIR patch 为：

```python
DRAFT_DIR = os.path.expanduser("~/Movies/JianyingPro/User Data/Projects/com.lveditor.draft")
```

这样生成的草稿文件直接落在剪映扫描目录，重启剪映就能看到。

## 重命名后必须同步修正 JSON

把目录从 `{draft_id}` 改名为 `{友好名字}-{后8位}` 后，
`draft_content.json` / `draft_info.json` 里还嵌着旧的 `/{draft_id}/` 绝对路径，
必须批量替换（`srt_to_draft.py` 已处理）：

```python
s = s.replace(f"/{draft_id}/", f"/{new_name}/")
```

## 剪映有"幽灵缓存"

如果以前在错误路径下生成过草稿，剪映会记住 → 即便删了目录仍显示"媒体丢失"：

1. **扫描数据库**：`~/Movies/JianyingPro/User Data/Cache/ILASDKDB/scan.db`
2. **.backup 目录**：每个草稿下的 `.backup/*.load.bak` 缓存了旧路径

彻底清理：
```bash
# 关闭剪映
lsof -ti:30000 | xargs kill -9 2>/dev/null
rm -f ~/Movies/JianyingPro/User\ Data/Cache/ILASDKDB/scan.db
# 删除有问题的草稿目录（含 .backup）
rm -rf ~/Movies/JianyingPro/User\ Data/Projects/com.lveditor.draft/有问题的草稿名
```
重启剪映会自动重建 scan.db。
